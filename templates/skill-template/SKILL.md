# Skill 名称

一两句话说明这个 skill 的定位：它是什么、不是什么。

## 触发与不触发

适合使用：

- 场景 1
- 场景 2

不适合使用：

- 相近但不该接的场景，以及该交给谁处理

## 工作流程

1. **第一步**：先判断 / 先确认什么（信息不全时先补什么信息，而不是直接给结论）
2. **第二步**：核心执行动作
3. **第三步**：收口与自检

细则材料按需加载，不要全部塞进主文件：

- 需要 X 时，读取 `references/<topic>.md`

## 行为边界

- 不支持什么（明确说出来，不要假装支持）
- 什么情况下先向用户确认再执行
- 失败 / 异常时怎么处理

## 输出约定

- 输出语言、固定结构或模板
- 交付时需要向用户说明什么（产物路径、保留的假设、待确认项）
