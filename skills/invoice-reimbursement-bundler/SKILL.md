# invoice-reimbursement-bundler

本地发票 PDF 的报销打包助手。只做确定性文件操作：读取 PDF 文本、解析发票号和价税合计、按规则组合、复制或移动文件。不依赖 OCR，不调用外部服务，所有文件操作默认以"先扫描预览、再按需执行"为准则。

## 触发与不触发

适合使用：

- "帮我统计一下这个目录的发票总额"
- "把这些发票凑成 3000 元报销"
- "按发票号查一下有没有重复"
- "把选中的发票移动到一个新文件夹"
- "从之前的报销目录和剩下的票重新拼一单"
- "/invoice-bundle" 或类似自然语言

不适合使用：

- 需要识别扫描件图片中的发票（只做文本提取，不做 OCR）
- 需要自动删除、清理旧目录或修改原始文件名
- 涉及增值税查验、真伪校验、税务申报等税务合规操作

## 工作流程

### Step 0 — 确认输入目录与目标金额

先确认：

- 输入目录路径（支持多个，用逗号分隔）
- 目标金额（例如 3000）
- 用户是要"只预览组合"还是"复制/移动文件"
- 若用户要求移动文件，再次确认后再执行 `--mode move`

### Step 1 — 扫描目录（脚本，无 LLM）

```bash
python3 {skill_dir}/scripts/invoice_bundle.py scan --inputs <dir>[,<dir> ...]
```

输出 JSON 包含：

- 解析出的发票明细（发票号、价税合计、路径）
- 目录发票总额
- 重复发票号及对应文件
- 无法解析的 PDF 文件列表

### Step 2 — 计算最优组合（脚本，无 LLM）

```bash
python3 {skill_dir}/scripts/invoice_bundle.py bundle --target <金额> --inputs <dir>[,<dir> ...]
```

组合规则：

- 总额 **不低于**目标金额
- 优先选择**超出金额最小**的组合
- 超出金额相同时，优先选择**张数最少**的组合
- 发票唯一性以**发票号**为准；重复发票号只保留首次出现的文件，其余计入重复列表

默认只输出组合方案，**不改动任何文件**。

### Step 3 — 执行复制或移动（如用户要求）

```bash
python3 {skill_dir}/scripts/invoice_bundle.py bundle --target <金额> --inputs <dir>[,<dir> ...] --output-root <dir> --apply --mode copy
```

- `--mode copy`：复制选中发票到结果目录，原文件保留
- `--mode move`：移动选中发票到结果目录；仅在用户明确要求移动时使用
- 结果目录命名：`报销{目标金额}_{选中总额}`，例如 `报销3000_3091.10`
- 执行后会再次扫描结果目录并校验合计金额，不一致时输出 `verify_ok: false`

### Step 4 — 交付与说明

向用户汇报：

- 扫描到的发票总数、总额、重复情况
- 是否找到满足目标的组合；选中发票张数、合计金额、超出金额
- 结果目录路径（若执行了 apply）
- 无法解析的文件清单，提醒用户手工处理

## 行为边界

- **默认只扫描和预览，不移动文件**。用户明确说"移动到文件夹"时才用 `--mode move`。
- **测试和验证一律使用复制出来的临时样本**，不要直接操作用户的原始发票目录。
- **发票唯一性以发票号为准**，不以文件名为准。
- **不做 OCR**：只支持文本可提取的 PDF 发票。无法提取文本的扫描件会被报告为无法解析，不会静默跳过。
- **不删除用户文件**：清理旧目录、删除 `.DS_Store` 等操作不纳入默认能力。
- **已有报销目录里的票如果仍存在，重新拼单时视为可继续使用**，除非用户明确排除该目录。
- 组合默认规则为 `总额 >= 目标金额`，优先超出金额最小，金额相同再优先张数少。如用户有其他偏好（例如严格等于、允许略低），先确认再调整。

## 输出约定

- 脚本输出为 JSON，便于后续步骤读取和校验。
- 向用户展示时使用中文，结构清晰：扫描结果 → 组合结果 → 执行结果 → 待确认/异常项。
- 涉及路径时同时给出绝对路径，方便用户核对。
