# 模板

以下模板是起点，不是强制填满的表格。生成内容时保持短、具体、项目特异。

## 最小 AGENTS.md

```md
# AGENTS.md

## 项目说明

- 包管理器和常用命令以 `README.md` 和 lockfile 为准。
- 修改业务行为前，先判断是否需要创建或更新 OpenSpec。
- 修改复杂模块前，如模块目录存在 `README.md`，先阅读该文件。

## 高风险区域

- 不要直接编辑生成代码；按对应生成流程更新。
- 修改请求层、权限层、路由层、埋点层、支付、审批流或核心配置前，需要说明影响范围和验证方式。

## Agent 扩展索引

- `.agents/README.md`：如存在，先阅读；用于索引项目本地 rules、skills、hooks、commands、templates、scripts 和 adapters。
- `.agents/` 只在确有项目本地 Agent 扩展材料时创建，不为空目录占位。

## 验证要求

- UI 行为变更至少运行受影响模块测试，或手工验证关键路径。
- 依赖、构建、请求、权限、部署相关变更需要补充验证结果。
```

## docs/README.md

```md
# docs

本目录存放非规格类长期辅助知识，不作为系统行为规格的主真源。

## 目录

- `context/`：项目背景、业务上下文、历史约束。
- `standards/`：项目特有工程规范。
- `runbooks/`：发布、回滚、排障和环境操作手册。
- `adr/`：长期架构决策记录。

系统行为、需求和验收标准在接入 OpenSpec 后进入 `openspec/`。
```

## 信息归位表

```md
| 信息 | 类型 | 来源 | 目标位置 | 可信度 | 原因 |
|---|---|---|---|---|---|
```

可信度：

- `已验证`：来自当前文件、命令输出或用户明确陈述。
- `推断`：根据上下文合理推断，但未直接确认。
- `待验证`：写入为事实前必须验证。
